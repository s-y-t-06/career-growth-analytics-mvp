# Model Card: Career Growth Churn Predictor

## Overview

This model predicts whether a newly signed-up user will churn, where churn is defined as having no `user_action` events between day 8 and day 21 after signup.

- **Task**: Binary classification (`is_churned`)
- **Target population**: Synthetic users of an AI career platform
- **Prediction horizon**: 7 days after signup (features); label window is days 8-21
- **Primary use case**: Prioritize Next Best Action interventions and quantify early churn risk

## Intended Use

- Score users at the end of their first week and feed the risk score into the rule-based Next Best Action engine.
- Compare onboarding variants by predicted churn risk.
- Generate interpretable feature importance for product and growth teams.

## Out-of-Scope Use

- This is a synthetic-data MVP; the model should not be deployed to a production product without retraining on real, consented data.
- The model must not be used to make automated decisions about individual users without human review and appropriate governance.

## Training Data

- Synthetic users generated by `career_growth.data_generation`.
- Features are derived only from events occurring before `signup_timestamp + 7 days`.
- Hidden propensity variables are never used as features.
- Categorical features: `acquisition_channel`, `country`, `device_type`, `user_intent_level`, `career_stage`, `onboarding_variant`, `language`, `timezone`.
- Numeric features: static temporal signals, core funnel indicators, and behavioral aggregates including `hours_to_first_action`, `hours_since_last_action_at_cutoff`, `unique_event_type_count`, `first_day_event_count`, `last_2_days_event_count`, `ai_assistant_interaction_count`, `job_detail_view_count`, `return_visit_count`.
- Time-based numeric features for users with no pre-cutoff actions are encoded as missing and imputed with the median in the modeling pipeline.
- Split: chronological 60% train / 20% validation / 20% test by `signup_timestamp`.

## Model Architecture

Two candidate models are trained and the best is selected by validation PR-AUC:

1. **Logistic Regression baseline**:
   - One-hot encoding for categorical features with infrequent category handling.
   - SimpleImputer (median) + StandardScaler for numeric features.
   - `class_weight="balanced"`.

2. **HistGradientBoostingClassifier**:
   - One-hot encoding for categorical features.
   - SimpleImputer (median) for numeric features.
   - Early stopping with `n_iter_no_change=10`.

## Evaluation

- **Selection metric**: PR-AUC on the validation set.
- **Threshold selection**: Optimized on the validation set using F1 score by default; Youden index is also supported.
- **Final evaluation**: computed exactly once on the held-out test set.
- **Metrics reported**: PR-AUC, ROC-AUC, log loss, Brier score, precision, recall, F1, accuracy, confusion matrix.
- **Calibration**: Reliability diagram using probability quantile bins.
- **Subgroup evaluation**: Metrics by `acquisition_channel`, `career_stage`, and `device_type`; small groups (sample size < 50) are flagged.

## Outputs

Saved to `artifacts/`:

- `churn_model.joblib` -- selected, fitted scikit-learn Pipeline
- `model_metadata.json` -- model name, version, timestamp, cutoff/window days, feature columns, threshold, split time ranges, sizes, churn rates, library versions
- `metrics.json` -- candidate validation metrics, selected model, threshold, validation metrics, test metrics, confusion matrix
- `feature_schema.json` -- correctly separated categorical and numeric feature lists
- `explainability.json` -- top coefficients, permutation importance, user explanations
- `user_explanations.json` -- at least 3 user-level explanations
- `subgroup_metrics.csv` / `subgroup_metrics.json` -- subgroup evaluation
- `nba_examples.csv` / `nba_examples.json` -- Next Best Action examples
- `plots/pr_curve.png`, `plots/roc_curve.png`, `plots/calibration.png`, `plots/confusion_matrix.png`, `plots/risk_distribution.png`, `plots/feature_importance.png`

## Explainability

- Logistic-regression coefficients are reported when the linear model is selected.
- Permutation importance (drop in PR-AUC) is computed on the validation set for both candidate models.
- Global top 15 features are saved and plotted.
- User-level explanations list the strongest positive and negative factors for individual predictions.
- **Important**: These explanations describe model associations, not causal effects.

## Limitations

- The underlying data is synthetic; causal relationships are calibrated for demonstration.
- The model has not been validated on real users or protected subgroups.
- Class imbalance is handled with `class_weight="balanced"` for the linear model and median imputation for missing numeric values.
- Subgroup metrics for small samples should be interpreted with caution.
